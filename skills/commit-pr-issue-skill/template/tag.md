## Format

**Tag**:
```markdown
v<major>.<minor>.<patch>
```

**Content**:
```markdown
<summary>

- <change 1>
- <change 2>
- <change 3>
```

## Reference

tag: use SemVer format, for example `v0.1.0`, `v1.2.3`
summary: a short release summary
body: required for release tags, describe the main changes and compatibility notes when applicable

## Example

**Tag**:
```markdown
v0.1.0
```

**Content**:
```markdown
发布提交、PR 与 Issue 模板规范

- 规范 commit、PR、issue、tag 的内容生成要求
- 移除 Issue 示例中的 GitHub CLI 创建命令
- 明确不写提交人、工具来源或 AI 署名
```
